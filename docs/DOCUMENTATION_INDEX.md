## Writing Documentation

When creating or updating documentation:

1. **Keep it organized** - Use clear headings and sections
2. **Use examples** - Show code examples when applicable
3. **Link appropriately** - Reference related documentation
4. **Be concise** - Explain clearly without unnecessary verbosity
5. **Update index** - Keep this file current
